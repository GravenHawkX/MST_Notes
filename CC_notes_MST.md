<p>Q21.  What are the 3 types of server virtualization ?<br>
Ans. The 3 types are :</p>
<ul>
<li>Full virt
<ul>
<li>Uses a special software : Hypervisor</li>
<li>It interacts directly with the physical servers CPU and disk space</li>
<li>Keeps each virtual server completely independent of the other virtual servers</li>
</ul>
</li>
<li>Para Virt :
<ul>
<li>Unlike full virt, guests systems are aware of each other.</li>
<li>The hypervisor here doesn’t need as much processing power to manage the guest OS’s</li>
</ul>
</li>
<li>OS-level virt
<ul>
<li>Doesn’t use a hypervisor at all, the virtualization capability is part of the Host OS.</li>
<li>All the Guest OS’s must be the same</li>
</ul>
</li>
<li>They all share a few common traits
<ul>
<li>The physical server is the host</li>
<li>The virtual servers are guests</li>
<li>These virt servers behave like physical machines</li>
</ul>
</li>
</ul>

<p>Q29. What is Service Oriented Architecture ?<br>
Ans. IBM made a model to show <mark>web services interaction</mark>, which consists of relationships between <mark>3 entities</mark>  :</p>
<ul>
<li><strong>Service Registry/Broker</strong> : Gives support for publishing and Locating services</li>
<li><strong>Service Provider</strong> : Provides e-business services or web services</li>
<li><strong>Service Requestor</strong> : Finds required services via the service broker</li>
</ul>
<p><mark>SOA</mark> is also a generic model describing <mark>service collaboration</mark>, not specific to web services</p>
<p>Operations in web service architecture / SOA :</p>
<ul>
<li><strong>Publish</strong> : Service descriptions need to be published for the service requestor to find them</li>
<li><strong>Find</strong> : Service requestor retrieves a service description or queries the service registry to find the required service.</li>
<li><strong>Bind</strong> : Service requestor invokes or initiates an interaction with the service.</li>
</ul>

<h2 id="cloud-computing">Cloud Computing</h2>
<p><em>It is a model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable resources.</em></p>
<h3 id="essential-characteristics-of-cloud-computing-">Essential characteristics of Cloud computing :</h3>
<ul>
<li>On-demand self-service :
<ul>
<li>A consumer can unilaterally provision computing capabilities, such as server time and network storage, as needed automatically without requiring human interaction.</li>
</ul>
</li>
<li>Broad Network access :
<ul>
<li>All the services provided are available over the network and accessed through standard mechanisms that promote use by heterogeneous thin or thick client platforms ( Mobile phones, laptops, tablets, etc )</li>
</ul>
</li>
<li>Resource Pooling
<ul>
<li>The providers computing resources are pooled to serve multiple consumers, with different physical and virtual resources dynamically assigned and reassigned according to consumer demand.</li>
</ul>
</li>
</ul>
